```
每个客户机首先通过socket将消息发送到服务器，然后服务器过滤并分发这些消息。实现了私人聊天和群聊。

从普通用户的角度介绍了项目设计的步骤。

首先，当服务器打开时，会在本地机器上创建一个ServerSocket对象，并启用一个线程来实现多个客户端和服务器之间的阻塞连接，服务器类本身会实现runnable接口来接收来自所有客户端的消息并向指定用户发送消息（包括不同消息发送实体的分类），并实现一个打印类，主动将服务器自身的消息发送给所有客户端（实现服务器控制台对用户的操作）。

接下来是客户。用户运行程序打开登录界面，通过输入预设的用户账号和密码连接到socket服务器，向服务器发送登录信号，将用户放入服务器的在线用户列表中，并将socket作为参数传递给friendlistwindown，然后传给MainWindow。用户在friendlistw Indow中选择的聊天对象将决定主窗口中显示的是哪个源消息，在关闭每个主窗口时，只会关闭当前窗口以保留相应的套接字，以便用户可以与其他不同的用户聊天，最后每个用户在退出响应位置时都会向服务器发送退出消息，销毁自己的在线列表，最终实现一个简单的聊天室。
```
